# Contract: Power Automate Webhook Payload

**Version**: 1.0
**Producer**: GitHub Actions (Python `power_automate.py` utility)
**Consumer**: Power Automate "When an HTTP request is received" flow

---

## Purpose

GitHub Actions sends vault file contents to the Power Automate webhook endpoint after each Jarvis run. Power Automate writes those files to SharePoint; OneDrive syncs them to the local Obsidian vault.

---

## HTTP Request

```
POST {POWER_AUTOMATE_WEBHOOK_URL}
Content-Type: application/json
```

---

## Payload Schema

```json
{
  "operation": "write_file",
  "files": [
    {
      "vault_path": "jarvis/digests/2026-06-14.md",
      "content": "# Nightly Digest: 2026-06-14\n\n..."
    },
    {
      "vault_path": "jarvis/tasks/task-001-gcp-discovery.md",
      "content": "# Task: GCP dataset discovery...\n\n..."
    },
    {
      "vault_path": "jarvis/knowledge/gcp/datasets.md",
      "content": "# GCP Datasets\n\n..."
    }
  ],
  "run_metadata": {
    "task_id": "task-001-gcp-discovery",
    "run_timestamp": "2026-06-14T02:39:22Z",
    "total_files": 3
  }
}
```

---

## Field Rules

| Field | Rule |
|-------|------|
| `operation` | Always `"write_file"` in MVP |
| `files[].vault_path` | Vault-relative path. Power Automate prepends the SharePoint library base path to construct the full SharePoint path. |
| `files[].content` | Raw UTF-8 markdown string. Not base64 encoded. Max 10MB per file (well within markdown file limits). |
| `run_metadata` | Informational only; Power Automate logs this but does not act on it. |

---

## Power Automate Flow Steps

1. **Trigger**: When an HTTP request is received (POST)
2. **Parse JSON**: Parse body using the schema above
3. **Apply to each** `files` item:
   - **Create file** (SharePoint connector): Site = `{SHAREPOINT_SITE_URL}`, Library = `{VAULT_LIBRARY}`, Folder = parent directory of `vault_path`, File name = filename from `vault_path`, File content = `content`
   - If file exists: **Update file content** (SharePoint connector) instead of Create
4. **Respond** with HTTP 200 OK

---

## Success / Error Responses

| Status | Meaning |
|--------|---------|
| 200 | All files written successfully |
| 400 | Malformed payload — GitHub Actions logs error and writes to task file |
| 429 | PA throttled — GitHub Actions retries with 30s backoff (max 3 retries) |
| 5xx | PA internal error — same retry logic as 429 |

---

## Security

- The webhook URL is stored as a GitHub Actions secret: `POWER_AUTOMATE_WEBHOOK_URL`
- The URL includes a unique token generated by Power Automate; it is not guessable
- No authentication header required (PA webhook URLs are secret-by-URL)
- No PII in payloads — vault files must not contain names, emails, or customer data before posting
