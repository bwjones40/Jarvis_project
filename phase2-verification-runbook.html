    #

    Purpose

    This runbook verifies that Phase 2 of the Jarvis project — the Agent Ecosystem Foundation — is fully functional and safe before it is relied on for nightly production runs. Working through it confirms five things:

    1. Phase 0 (Stabilization) — the agents now make real Anthropic API calls (no more zero-token synthetic output), unit tests gate every CI run, task IDs are unique, concurrent runs queue instead of cancelling, PII protection runs in standard mode, the Node.js 20 deprecation warning is gone, and Power Automate upserts (one file per task, not duplicates).
    2. Sprint 1 (Structured Logging) — every run writes a schema-valid JSON log to SharePoint with per-agent timing, token, and cost data.
    3. Sprint 2 (Validation Agent) — research and writer outputs are scored, and the retry / skip / escalate logic behaves correctly, including when the validator itself crashes.
    4. Sprint 3 (Monitoring) — the morning digest shows a per-run quality table, and the bi-weekly stats report aggregates trends across runs with correct analysis windows.
    5. No Phase 1 regressions — the original digest, task record, and vault outputs still work exactly as before.

    Each step below tells you exactly what to do, what success looks like, what evidence to capture, and how an AI reviewer can later confirm you did it correctly. You do not need to already know how to perform any step — every action is spelled out.

print(f"PASS: {len(files)} log file(s) found and valid")
